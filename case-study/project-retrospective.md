# 项目复盘 · leonardo-chinese-exhibition

> 写给自己，也写给将来想做数字策展的人。

---

## 一 · 做对了什么

### 1.1 学术研究 → 数字策展 → 作品化的三段路径

最关键的决策不是"做得漂亮"，而是把**项目分成三层独立但互锁的产物**：

1. **学术翻译层**（v1.0 → v1.5）——把意 / 英语学术内容转译成中文叙事
2. **数字展览层**（v1.5 → v1.8）——把转译内容做成"真展览"而非"长文"
3. **作品化层**（v1.9 → v2.0）——把独立项目做成"可引用、可被外部工程验证的案例"

这三层可以独立推进，但每一层做完才能进下一层。如果一上来就想"做作品集案例"，结果就是"看起来完整、实际空心"。

### 1.2 公共域先行的版权策略

所有真实手稿图像都从 Wikimedia Commons 公共域拿，而不是去爬温莎皇家收藏的图库。

这看上去"保守"，实际是**最聪明的策略**：

- 不需要担心爬虫被封 / 二次发布权利不清晰
- 每张图像都有可引用的 contributor 链
- 即使网站被质疑，证据链是"我们用的是公共域"

**未来如果要做商业版本**，换授权链路是另一件事；但 v1.0 - v1.9 阶段所有图像都可商用引用、CC BY 兼容。

### 1.3 版本化 + 报告同步

每一步都强制留 commit + 报告：

- 14 份版本/审计报告（reports/leonardo_*_v*.md）
- 4 层 marker（v1.5b / v1.7 / v1.8 / v1.9 在 HTML 里都活）
- GitHub Actions 每次 push 都审计

**这意味着任何中间状态都可被外部工程拉到、可被未来自己对比、也可被雇主/合作方审查。**

### 1.4 8 节结构的稳定性

虽然展览做了 14 个版本，但**主体叙事结构（8 节标题）从 v1.0 起几乎没变**：

- 序厅：被拆散的思想博物馆
- 1 · 纸页的命运
- 2 · 《大西洋手稿》
- 3 · 温莎的绘图
- 4 · 同一张纸上的艺术与科学
- 5 · 水印与纸张
- 6 · 复原拼合
- 7 · 平台如何工作
- 8 · 达·芬奇方法

每版的差异是**视觉/图像层**，不是**结构层**。结构稳定 → 内容演化可以持续 5 年而不需要推倒重写。

## 二 · 中间踩了哪些坑

### 2.1 最初几个版本"不像展览"

**v0.x - v1.4 的状态**：是排版好的长文 + 几张简陋 SVG。视觉上"博客长文感"很重。

为什么？——我把"内容写完"当作完成标志，没意识到：

- **真实展览**有"图 + 卡 + 视觉锚"三件套，长文只有一项
- **视觉一致性**比"好看"重要，要让每一节看上去是一个展览里的同一节
- **展品索引 / 地图**是展览感的关键 banner，不是装饰

**踩坑教训**：做完内容不等于做完展览。要问的不是"我有没有讲清楚"，而是"我有没有图、没有卡、能不能让观众 1 秒看见?这是不是展览?"

### 2.2 占位提示语忘了删

`site/index.html` footer 有一句临时占位说明："图片使用占位与结构图，实际使用时请替换为真实手稿影像。"

这句话在 v1.7 之前一直都在。**直到 v1.7（exhibit image upgrade）才删除**。

**踩坑教训**：临时占位 / "TODO" / placeholder 文本应该在每个版本末期自己检查一次。它们很容易跑过 review 因为"反正下一步会删"——但实际上"下一步"经常跟下一个迭代一起远去。建议在每次 release 报告里**显式 grep** 一次 "TODO / FIXME / 占位 / TODO"。

### 2.3 RCIN 编号没核对

v1.7 报告里我用了"近似"RCIN 值（912695 / 919023 / 912377）。

实际上 Wikimedia 上能查到的真实 RCIN 是 912310 / 919003 / 912363。

**直到 v1.8 报告对照原始 API 数据才纠正**。

**踩坑教训**：**不要再做"差不多"的标识符**。 RCIN、DOI、ISBN、文物编号这种"权威编号"必须是**可双向往返**的值——一个 RCIN 应该能让你回到原馆藏页面、能让你在 Wikimedia 找到对应文件、能让你跟原研究者交叉验证。

### 2.4 文件权限 0600 是个隐形坑

30 个文件全是 `0600`（仅 root 可读），但 git index 只跟踪 executable bit、不跟踪 0600/0644 差异。

**v1.5c hygiene 修复时才显式 chmod 0644**。

**踩坑教训**：当文件被传到 GitHub Actions runner 上时，runner 用户可能读不了 0600 文件（因为 owner 不同）。**一定要在 release 前做一次 `ls -la` + `chmod 0644`**。把"权限 hygiene"作为 checklist 的一项。

### 2.5 favicon 与 og-cover 没挂上

`assets/favicon.svg` 和 `assets/og-cover.svg` 两个文件 v1.0 起就在仓库里。但 `<head>` 里**没有 `<link rel="icon">` 也没有 `og:image>`**——这两个资源从未被引用。

**直到 v1.9 final polish 才补上 4 行 `<head>`**。

**踩坑教训**：资源 ≠ 引用。**做完资源不等于用上资源**。每次有 "static asset" 上线前要 grep 一次"它在 page 里被引用了吗？"——尤其 favicon 这种"以为默认就有"的资产。

## 三 · 为什么早期页面不像展览

更深地问一次：**为什么我自己在 v0.x - v1.4 时没有意识到"不像展览"？**

答案可能是：

1. **没有用过真实展览做参考**
   我大部分参考是博物馆微信公众号推文、个人博客、网络文章。这些都不"像展览"——它们是"图文结合的长文"。
   真正的视觉参考应该是：博物馆在线展厅、OpenAI 官网、Stripe / Linear 等的 editorial 风产品页、Notion 的 editorial 风格。

2. **没有 8 节 × 4 件套的模板意识**
   每个展览节都是一个**可复用的结构单元**。我没有意识到这一点，所以每节有自己不同的长相（有的有 figure、有的有 ul、有的什么都没有）。参展者感觉不到"统一节奏"。

3. **把"做完内容"当 endpoint**
   内容是展览的一半，**视觉一致性才是另一半**。先有视觉一致性，然后内容填进去才有节奏。

**v1.7 做对了事情**：把每节都强制为 `section-content + figure + meta-cards + viewing-guide` 四件套。

### 3.1 关键转折点

我整理过心智上的几个转折：

| 转折点 | 之前 | 之后 |
|---|---|---|
| v1.5 redesign | "做完内容" | "OpenAI editorial 风" |
| v1.7 image upgrade | "加图就好" | "每节 4 件套结构 + 展品索引" |
| v1.8 real image | "自己能画 SVG" | "公共域 + 真实手稿像素" |
| v1.9 final polish | "上线就行" | "资产审计 + credit 一致" |
| **v2.0 portfolio case** | "作品集=发页面" | "作品集=完整可引用的工作流案例" |

每次转折都对应一个**明确的内部否定**："上一个思路不够"——而不是"再加一些东西"。

## 四 · v1.7 / v1.8 怎么解决问题

### v1.7 解决的问题

**问题**：v1.5 redesign 之后的页面像是"博客长文"，还不是展览。

**v1.7 解决路径**：

1. **加展品索引** — `#exhibit-index` 板块，8 张卡片作为"展览封面"
2. **每节 4 件套** — section-content / figure / meta-cards / viewing-guide，全部 8 节统一
3. **加原创 SVG 图解** — 4 个全新（collection-split / watermark / recomposition / platform-tool-wall）+ 1 个升级版 manuscript-journey
4. **删除占位文案** — footer 不再写"请替换为真实手稿"
5. **移动端单列** — image grid、gallery 等移动单列
6. **从 13 KB → 33 KB** — 视觉上"扩展"、内容上"精炼"

### v1.8 解决的问题

**问题**：v1.7 仍然只用原创 SVG 框图，**没有真实手稿像素**——这是"假展览"最核心的"假"。

**v1.8 解决路径**：

1. **从 Wikimedia Commons 下载 6 张真实公共域手稿**（温莎 4 + Codex 2）
2. **每张都标注 RCIN / folio / 年份 / 馆藏 / credit-line**
3. **展品索引 4 张卡片 + section-3 画廊 4 张 + section-2 展品柜 2 张**——共 10 个真实图像渲染位
4. **3 张 `.image-placeholder-pro` 占位卡**标记 B1 / B4 / B5 平台截图候选（v1.9/v2.0 补）
5. **修正 3 个 v1.7 报告里的 RCIN 编号错误**

**v1.7 → v1.8 是质变。** 在此之前是"自制图解展览"，之后是"真实手稿 + 自制图解双驱动"。

## 五 · 对未来数字展览项目的启发

### 5.1 通用方法

1. **结构先于内容**：先把节数、节标题、节核心问题定下来，再填正文。
2. **图解先于真实图**：先用自制 SVG 把逻辑画清楚，再用真实图像替换。
3. **公共域永远是退路**：版权清晰、可商用、可引用的图像是基石。
4. **报告与代码同仓**：每一步都留报告。**报告 = 版本控制 + 历史考古 + 雇主审查**。
5. **三层 marker 工程可见性**：`<meta>` + HTML 注释 + footer，写过之后可被外部 `curl + grep` 验证。

### 5.2 给"独立策展者"的建议

- **不要一次性做完所有事**——按 "学术翻译 → 视觉升级 → 真实图 → 作品化"四步。
- **每步留 commit + 报告**。
- **每步都有一个"独立可用"的端点**——v1.5 已经是像样的页面，v1.7 已经像展览，v1.8 已经嵌了真实手稿。
- **把"完成" 视为连续概念**——v1.0 既是开始也是完成（上线），v1.5 也是完成（视觉成熟），v1.8 也是完成（真实图），v2.0 也是完成（可被引用）。
- **不要小看 README 与 posts / case-study**——独立项目如果不上传到工作流入口（README / 个人主页 / X / 作品集），就是无形资产。

### 5.3 给"想做'中文 + 国际'展览"的建议

- **保留学术专有词**（Leonardo//thek@、Codex Atlanticus、RCIN）——它们是研究者的入口
- **中文叙事，国际关键标识符**——双语并行
- **每个图、每个数据、每个引用都可被双向往返**——这是"可被外部验证"的实质
