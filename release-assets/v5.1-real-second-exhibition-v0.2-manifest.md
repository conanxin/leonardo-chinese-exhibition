## Rollback principle

- this release is **content-additive** relative to
  `v5.0-real-second-exhibition-deployment`: the v0.2 content is
  a strict superset of v0.1 in the same file paths.
- to roll back to v0.1: revert the three chain commits
  (`f43c5479`, `fff6e78`, `96bef6a`); the v0.1 source remains
  intact in the v5.0 tag target `ac0f19e2c03b…`.
- to roll forward from v5.1: the v5.0 tag remains as the
  initial-deployment anchor; the v5.1 tag becomes the
  v0.2-content anchor.
- this release is **not** a security-sensitive change (no new
  data, no new external request, no new visitor-side script,
  no workflow change).

## Tag target rule (this release)

- the new annotated tag `v5.1-real-second-exhibition-v0.2`
  targets the **freeze commit** (the commit that contains this
  manifest).
- the **backfill commit** (which records the actual tag object
  SHA, the actual Release URL, the actual `publishedAt` timestamp,
  and any post-tag evidence) is **not** the tag target.
- the previous stable tag `v5.0-real-second-exhibition-deployment`
