# 中文数字展览内容写作指南

本指南讲解从标题到 glossary 的每个模块应该怎么写。
本指南配合 `_template/data/*.example.json` 使用，先看 example 文件再看本指南会更顺。

---

## 展览标题

展览标题是观众第一眼看到的判断标准。写好标题要做到：

- 说明展览对象或问题（"X 的 Y 种读法"、"Z 的早期演变"）
- 避免只有抽象概念（"意义"、"启示"、"再思考"等空词不构成标题）
- 避免过度营销语（"震撼"、"颠覆"、"必看"）

模板字段：`exhibition.example.json` → `title`

---

## Hero 文案

Hero 是展览首页最重要的导语。

要求：

- 说明展览在问什么
- 不堆概念
- 不做夸张断言
- 100–200 字为宜

模板字段：`exhibition.example.json` → `hero.lede`

错误示范：

- "穿越时空的视觉盛宴"（营销腔）
- "X 的前世今生与未来"（抽象 + 空话）

正确示范：

- "本展览考察 X 在 1890–1930 年间的四种版本，及其背后的观众、技术与机构差异。"
- "我们试图回答一个问题：当 X 被搬到 Y 上时，它失去了什么、保留了什么？"

## 3 分钟导览

3 分钟导览是给"第一次打开页面的观众"的快速导航。

格式：

- 5 点以内
- 每点一句
- 每句 20–40 字
- 第一句说明展览对象，第二句说明核心问题，后续为分段导引

模板字段：`exhibition.example.json` → `quick_tour.items[]`

每点都应回答"如果观众只能记住一件事，ta 应该带走什么"。

## Section 写作

每个 section 是一个独立叙事单元。

模板字段：`sections.example.json` → `sections[]`

每个 section 建议包含：

- **kicker**：section 的角色名（"第一章"、"早期形态"、"观众如何接触"）
- **title**：section 标题，15 字以内为宜
- **body**：section 正文，200–500 字
- **takeaway**：section 想让观众带走的核心判断（一句话）
- **viewer_action**：观众在 section 末尾的引导动作（"继续阅读下一节"、"展开 artifact card"、"返回展览地图"）

避免：

- body 内堆引用编号（读者不会回头看）
- takeaway 写成与 title 重复的句子
- viewer_action 写成"无"

## Artifact card 写作

展品卡是展览的最小叙事单元。

每个 artifact card 包含：

- **title**：展品名或展品主题
- **image**：图片路径（必须是 `assets/` 内已 audit 的图）
- **alt**：图片 alt text（描述画面内容）
- **caption**：图注（一句话，告诉观众这张图是什么）
- **source note**：来源说明（必须包含机构 + URL + 标识符）
- **credit line**：权利说明（是否允许复用，回到原页确认）
- **viewing note**：观看建议（30–80 字，引导观众看哪里、为什么）

模板字段：`sections.example.json` → `sections[].artifacts[]`

特别注意：

- alt 描述画面，caption 描述展品，二者不混用
- source note 与 credit line 是两块内容，不互相替代
- viewing note 不要重复 caption 的内容

## Deep block 写作

Deep block 用于在 section 内展开某一类深入内容。本模板提供四类：

### 1. deep-reading block

适用：文本细读、文段对照、术语辨析。

写作原则：

- 选 1–3 段核心文本
- 每段配一段说明
- 结尾给一句"读完这段，读者应该意识到"

### 2. material-evidence block

适用：材料证据、版本对比、流传考证。

写作原则：

- 选 2–4 件材料
- 说明每件材料的形态、来源、年代
- 末尾给一句"这些证据说明什么"

### 3. visual-thinking block

适用：图像细看、构图分析、观看路径。

写作原则：

- 描述画面具体内容（不是"美"或"震撼"）
- 说明为什么这个细节重要
- 不下断言，给读者留判断空间

### 4. research-model block

适用：研究方法、采集流程、分类逻辑。

写作原则：

- 说明方法是什么
- 说明方法的边界
- 不假装方法是唯一正解

## Glossary 写作

Glossary 是给第一次接触这个领域的观众准备的术语表。

每个术语：

- 50–100 字
- 解释观众为什么需要知道（不只是定义）
- 不写成百科词条
- 避免只堆定义

模板字段：`glossary.example.json` → `terms[]`

字段说明：

- **term**：术语
- **definition**：定义（不是词典定义，而是"在这个展览里这个术语意味着什么"）
- **why_care**：观众为什么需要知道（"如果你不知道 X，下一节会看不懂"）

错误示范：

- "透视：一种绘画技法。"（这是词典定义，没告诉观众为什么要学）

正确示范：

- "透视：让二维平面看起来像三维空间的方法。在本展览第三节，画家改变透视点的位置，让观众从不同角度看见同一座城市。"

---

## 写作节奏建议

- 每个 section 控制在 300–600 字
- 整场展览控制在 2500–5000 字
- 正文里少用 emoji 符号，多用结构（标题、列表、表格）
- 不要把 markdown 当成设计工具，结构服务于阅读
