# 来源与权利检查清单

本清单用于在展览发布前确认所有图片来源、平台截图、自制图解的权利边界。
本清单基于 v2.9-real-source-rights-audit 阶段验证，配合 SOURCE_AUDIT_MANIFEST.md 和 RIGHTS_AND_SOURCES.md 使用。

---

## 每张图片必须检查

对每张进入 `assets/` 的图片，按以下清单逐项确认：

- [ ] **文件路径**：在 `assets/` 下，相对路径正确
- [ ] **来源机构**：博物馆 / 图书馆 / 档案馆 / 个人收藏 等
- [ ] **来源 URL**：可以回到原页面的 URL
- [ ] **标识符 / 编号**：馆藏编号、catalog ID、ISBN、DOI 等
- [ ] **alt text**：描述画面内容（不是来源描述）
- [ ] **caption**：图注，告诉观众这张图是什么
- [ ] **source note**：来源说明（机构 + URL + 编号）
- [ ] **credit line**：权利说明（机构要求的署名方式）
- [ ] **是否用于 lightbox**：放大版本是否单独文件
- [ ] **data-credit 是否存在**：在 `data/assets.json` 中是否登记
- [ ] **是否允许复用**：license 类型（PD / CC0 / CC-BY / 仅限研究使用 / 馆藏限定）
- [ ] **是否需要回到原机构确认**：截图日期、访问路径、当时的页面状态

未通过以上任意一项，禁止加入 `assets/`。

---

## 平台截图检查

对每张来自外部平台（数字馆藏、数据库、研究工具）的截图：

- [ ] **平台名称**：来源平台（如 IIIF、Internet Archive、Museo Galileo 等）
- [ ] **截图来源 URL**：访问该截图时的具体页面 URL
- [ ] **截图日期**：截图访问的日期（YYYY-MM-DD）
- [ ] **模块说明**：截图中展示的是什么功能 / 视图 / 文档
- [ ] **截图是否用于评论 / 研究说明**：明确说明截图用于评论该平台的展示方式，不作为原始馆藏图像
- [ ] **是否避免伪装成原始馆藏图像**：截图本身是"关于原始资料的工具视图"，不是原始资料本身，必须在图注和 source note 中说明这一点

特别注意：

- 平台截图的 caption 必须包含"平台界面"或类似措辞
- 平台截图不允许作为展览主视觉
- 平台截图必须说明平台的服务条款（即使截图本身是公开可见的）

---

## 自制图解检查

对每张 project-generated 图（自制 SVG、流程图、关系图）：

- [ ] **是否明确 project-generated**：在 alt text 与 caption 中明确"自制图解"或"作者绘制"
- [ ] **是否说明依据**：列出依据哪些原始资料或数据
- [ ] **是否避免伪装成原始资料**：不放在原始资料的位置，必须有"自制"标识
- [ ] **是否需要单独 LICENSE**：如果是 CC-BY-SA 等协议下发布，需在 LICENSE 中明确归属

特别注意：

- 自制图解不能引用未授权的原始图片作为底图
- 自制图解可以引用公开数据，但数据来源必须列出
- 自制图解在视觉上必须与原始资料明显区分（不同色板、不同边框、不同 alt）

---

## 发布前检查

每次 release / freeze 之前：

- [ ] **SOURCE_AUDIT_MANIFEST.md 是否完整**：每张图片都登记
- [ ] **RIGHTS_AND_SOURCES.md 是否完整**：所有来源、license、署名要求都登记
- [ ] **README 是否链接 source docs**：主 README 指向 source docs
- [ ] **release notes 是否说明权利边界**：本次 release 的权利声明
- [ ] **新增图片是否重新 audit**：freeze 之间新增的图片都重新走清单

## 免责声明

本 rights 文档不是法律意见。

- 文档记录的是本次展览作者团队对来源与权利的核验过程与判断
- 文档不构成对外部复用者的授权
- 如果第三方希望复用本展览的内容，应回到原始机构页面确认该机构当下的许可状态
- 权利状态可能随时间变化，发布时的"已确认"不等于永久授权
- 任何权利纠纷以原机构的页面与许可条款为准
